# Xvideo 对标 Animeko 开发计划

更新日期：2026-06-06

## 目标

将 Xvideo 从“资源站影视浏览器”推进为接近 Animeko 的“找番、追番、看番一体化”桌面客户端。功能和布局参考 open-ani/animeko，但明确不做账户登录、Bangumi 登录、云同步和需要用户账号体系的能力。

测试资源统一使用：

```text
https://cj.lziapi.com/api.php/provide/vod/at/xml/
```

## 范围边界

### 必须对标

- 找番：主页发现、分类、搜索、标签筛选、时间表。
- 追番：收藏、继续观看、观看进度、已看/当前/下一集状态。
- 看番：详情页、播放源选择、剧集列表、播放器、上一集/下一集、播放失败降级。
- 数据源：多数据源管理、源健康状态、播放源自动选择。
- 离线：下载/缓存任务页、任务状态、失败重试、文件管理。
- 弹幕：播放器弹幕层、开关和基础显示设置。
- 设置：数据源、播放器、弹幕、缓存、本地数据导入导出。
- 大屏/PC 布局：左侧主导航、首页分区、找番页、时间表、详情页、播放器、缓存页、设置页。

### 不做或本地替代

- 不做账户登录。
- 不做 Bangumi 云同步。
- 不做用户云收藏。
- 云端追番进度改成本地观看进度。
- 账号相关设置入口不出现；需要同步的地方提供本地导入/导出替代。

## 当前基线

已具备：

- XML/JSON/扁平 XML 数据源解析。
- 数据源添加、测试、切换和删除。
- 最新更新、分类、关键词搜索、年份/地区筛选。
- 海报、简介、演员、导演、年份、地区、评分、播放列表详情。
- 内嵌播放器、播放窗口、上一集/下一集、15 秒快进/快退。
- 本地收藏，收藏绑定数据源。
- 本地继续观看记录，支持恢复剧集和时间点。
- mp4 直链下载和下载架子。

主要缺口：

- 信息架构仍偏“影视库”，不是 Animeko 式追番应用。
- 没有时间表页。
- 没有标签搜索页和多标签筛选。
- 下载还不是完整离线缓存管理。
- 没有弹幕层和弹幕设置。
- 设置入口不完整。
- 播放源选择器和自动选源还不够 Animeko 化。

## 开发流程

每个阶段都按同一套流程推进：

1. Inspect
   - `git status -sb`
   - 确认分支、远端、最近提交。
   - 保留无关未跟踪或用户改动。
2. Implement
   - 只改本阶段相关文件。
   - SwiftUI/AppKit 沿用现有项目结构。
   - 手工编辑使用 `apply_patch`。
3. Build
   - 代码变更后运行 `swift build`。
   - 修完所有编译错误再继续。
4. App Test
   - UI、播放、下载、数据源相关变更运行 `Scripts/build_app.sh`。
   - 启动 `.build/app/Xvideo.app` 测真实 macOS 工作流。
5. Documentation
   - 用户可见功能变更同步 `README.md` 和 `README.en.md`。
   - 架构或计划变化同步 `Docs/`。
6. Commit and Push
   - 只 stage 本阶段相关文件。
   - 本地提交，提交信息清楚描述功能。
   - 推送到 GitHub。
7. Release
   - App 行为或代码变更需要打包 release。
   - tag 使用日期：`YYYY-MM-DD`。
   - 资产名：`Xvideo-YYYY-MM-DD-macOS.zip`。
   - release notes 顺序固定：`中文`、`English`、`Verification`、`Build`。
   - release notes 包含资产名和 SHA-256。

## 阶段计划

### Phase 1：信息架构和主布局对齐

目标：先把应用骨架改成 Animeko 式追番客户端，后续功能都有稳定入口。

开发内容：

- 重排左侧主导航：
  - 首页
  - 找番
  - 时间表
  - 继续观看
  - 我的收藏
  - 离线缓存
  - 设置
- 分类从主导航移入“找番”页，避免侧边栏被源分类占满。
- 数据源状态放到底部状态区，显示当前源、格式、健康状态。
- 首页改成分区流：
  - 继续观看
  - 今日/本周更新
  - 最近更新
  - 标签入口
  - 分类推荐
- 保留现有影院视觉，但降低“电影网站 hero”占比，使首页更像追番 dashboard。

涉及文件：

- `Sources/Xvideo/Presentation/Views/ContentView.swift`
- `Sources/Xvideo/Presentation/Views/CinematicSidebarView.swift`
- `Sources/Xvideo/Presentation/Views/MediaBrowserView.swift`
- 新增或拆分：首页、找番、时间表、缓存、设置相关 view 文件。

测试标准：

- 最小窗口 `1280x780` 不重叠。
- 侧边栏所有入口可切换。
- 首页各分区有加载、空状态、错误状态。
- 当前已有浏览、收藏、继续观看入口不回退。

验收标准：

- 打开 App 第一屏能看出是追番应用，而不是普通影视库。
- 入口结构和 Animeko PC 端主要模块一致，排除账号入口。

### Phase 2：本地索引和测试源适配

目标：基于 XML 测试源建立可复用的本地聚合索引，为时间表、标签和发现页服务。

开发内容：

- 从 `VodItem` 聚合字段：
  - 分类：`type_name`
  - 标签：`vod_class`
  - 年份：`vod_year`
  - 地区：`vod_area`
  - 语言：`vod_lang`
  - 更新时间：`vod_time`
  - 更新备注：`vod_remarks`
- 建立本地 `DiscoveryIndex` 或等价 view model。
- 对缺字段做降级：
  - 没有更新时间时放入“最近更新”。
  - 没有标签时使用分类名兜底。
  - 没有地区/年份时不显示空标签。
- 数据源测试结果增加可读状态：
  - 分类数
  - 条目数
  - 最近测试时间
  - 失败原因

涉及文件：

- `Sources/Xvideo/Domain/Models/VodModels.swift`
- `Sources/Xvideo/Presentation/ViewModels/LibraryViewModel.swift`
- `Sources/Xvideo/Data/Repositories/DefaultLibraryRepository.swift`
- `Sources/Xvideo/Presentation/Views/SourceManagerView.swift`

测试标准：

- 使用指定 XML 源能生成分类、标签、年份、地区、更新时间索引。
- 空字段、HTML 简介、异常数字字段不崩溃。
- 搜索不支持时仍能保留浏览能力。

验收标准：

- 后续页面不再各自重复解析标签/时间，统一从本地索引取数据。

### Phase 3：找番页和标签搜索

目标：对标 Animeko 的“丰富检索方式：标签搜索”。

开发内容：

- 新增“找番”页。
- 顶部搜索框支持标题、演员、关键词。
- 标签筛选区：
  - 类型
  - 年份
  - 地区
  - 语言
  - 标签
- 支持多条件组合筛选。
- 搜索结果支持：
  - 海报网格
  - 紧凑列表
  - 右侧/悬浮详情预览
- 空结果显示明确提示。

涉及文件：

- 新增 `DiscoveryView.swift`
- 新增 `TagFilterView.swift`
- 复用或抽取 `MoviePosterCard`
- `LibraryViewModel` 增加筛选状态和查询入口

测试标准：

- 标题搜索可用。
- 单标签筛选可用。
- 多标签组合可用。
- 清空筛选恢复结果。
- 空结果不崩溃。
- 从结果可进入详情和播放。

验收标准：

- 找番页能完成“搜索、筛选、看详情、播放”的闭环。
- 布局接近 Animeko PC 搜索/详情体验。

### Phase 4：新番时间表

目标：对标 Animeko 的“新番时间表”。测试源不是 Bangumi，所以时间表使用资源站更新时间做本地替代。

开发内容：

- 新增“时间表”页。
- 日期维度：
  - 今日更新
  - 最近 7 天
  - 本周分组
- UI 形态：
  - 顶部日期/星期 tabs
  - 每日更新卡片列表
  - 卡片显示标题、封面、更新集数、分类、更新时间
- 没有更新时间的内容进入“未标明时间”分组。

涉及文件：

- 新增 `ScheduleView.swift`
- 本地索引模型补充日期分组

测试标准：

- 指定 XML 源能展示最近更新时间。
- 日期切换不丢状态。
- 无时间字段内容可降级显示。
- 点击卡片进入详情。

验收标准：

- 时间表页即使没有 Bangumi 精确放送表，也能提供可用的资源站更新表。

### Phase 5：详情页和播放源选择器对齐

目标：将详情页从“影视详情”升级成 Animeko 式番剧详情和媒体选择。

开发内容：

- 顶部详情区域：
  - 背景/封面
  - 标题
  - 评分
  - 标签
  - 年份/地区/语言
  - 追番状态
  - 继续播放按钮
- 剧集列表：
  - 当前集
  - 已看标记
  - 下一集建议
  - 更新状态
- 播放源选择器：
  - 简洁模式
  - 详细模式
  - 源可用性标识
  - 失败时自动尝试下一个播放源

涉及文件：

- `Sources/Xvideo/Presentation/Views/MovieDetailView.swift`
- `Sources/Xvideo/Domain/Services/SourceParser.swift`
- `Sources/Xvideo/Domain/Services/PlaybackURLResolver.swift`
- `Sources/Xvideo/Presentation/Views/PlayerViews.swift`

测试标准：

- 多播放源影片可切换。
- 无播放源显示空状态。
- 播放失败不崩溃。
- 已看和继续观看状态同步。

验收标准：

- 详情页能清楚回答：这是什么、看到哪、下一集看哪、用哪个源看。

### Phase 6：播放器体验和弹幕基础

目标：对标 Animeko 的看番播放器体验，先做本地/基础弹幕，不引入账号。

开发内容：

- 播放器工具栏：
  - 剧集列表
  - 播放源
  - 弹幕开关
  - 缓存按钮
  - 设置按钮
- 弹幕基础层：
  - 开关
  - 字号
  - 透明度
  - 速度
  - 显示区域
- 支持本地弹幕文件导入作为第一阶段能力。
- 无弹幕时正常播放，显示轻量提示。

涉及文件：

- `Sources/Xvideo/Presentation/Views/PlayerViews.swift`
- 新增 `DanmakuOverlayView.swift`
- 新增弹幕设置存储

测试标准：

- 弹幕开关不会影响播放。
- 弹幕层不遮挡播放器控制。
- 全屏窗口和内嵌播放器表现一致。
- 无弹幕、弹幕文件解析失败都不崩溃。

验收标准：

- 播放器具备 Animeko 式“可看弹幕番”的基础体验。

### Phase 7：离线缓存页

目标：从下载架子升级到 Animeko 式离线缓存管理。

开发内容：

- 新增“离线缓存”页。
- 任务分组：
  - 全部
  - 下载中
  - 已完成
  - 失败
- 按影片/剧集分组展示。
- 操作：
  - 暂停
  - 取消
  - 重试
  - 删除
  - Finder 中显示
- 对不支持缓存的资源显示原因。
- m3u8 缓存作为独立能力评估；未实现前 UI 明确“当前仅支持直链资源缓存”。

涉及文件：

- `Sources/Xvideo/Infrastructure/Downloads/DownloadManager.swift`
- 新增 `OfflineCacheView.swift`
- 新增缓存任务持久化

测试标准：

- mp4 下载显示进度。
- 完成后可显示文件。
- 失败任务有原因。
- 删除任务清理 UI 和本地文件。
- 不支持的播放地址不崩溃。

验收标准：

- 用户能从“离线缓存”页管理缓存，而不是只能看右下角下载架子。

### Phase 8：设置和本地数据管理

目标：对标 Animeko 的个性设置，用本地能力替代账号同步。

开发内容：

- 新增设置入口或设置窗口。
- 设置分区：
  - 数据源
  - 播放器
  - 弹幕
  - 缓存
  - 外观
  - 本地数据
- 本地数据：
  - 导出收藏、观看记录、数据源配置
  - 导入本地备份
  - 清空观看记录
  - 清空缓存
- 设置持久化。

涉及文件：

- 新增 `SettingsView.swift`
- 新增偏好存储模型
- 接入 `XVideoApp.swift` commands 或主界面入口

测试标准：

- 设置修改后立即生效或按说明重启生效。
- 重启后设置仍保留。
- 导入导出文件格式可读且失败可提示。

验收标准：

- 无账号情况下，用户仍可管理自己的本地数据和观看体验。

### Phase 9：视觉统一和响应式验收

目标：完成布局对齐和视觉 polish。

开发内容：

- 统一导航密度、卡片圆角、间距、标题层级。
- 统一空状态、加载状态、错误状态。
- 检查所有按钮图标和 tooltip。
- 大屏优先，同时保证最小窗口可用。
- 截图更新 README 预览图。

测试视口：

- `1280x780`
- `1440x900`
- `1728x1117`
- 宽屏外接显示器

测试标准：

- 无重叠。
- 无按钮文字溢出。
- 海报比例稳定。
- 图片失败不造成布局跳动。
- 切换页面无明显闪烁。

验收标准：

- 主界面、找番、时间表、详情、播放器、缓存、设置都能看出 Animeko PC 端参考痕迹。
