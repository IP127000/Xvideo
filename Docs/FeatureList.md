# Xvideo Web 功能清单

本文基于当前 `web` 分支代码整理，重点说明“已经实现了什么”“现在怎么操作”“当前版本的边界在哪里”。

## 产品定位

- `web` 分支的 Xvideo 是一个浏览器端影视客户端，核心目标是让用户接入自己的媒体 API 后，在 Web 端完成浏览、搜索、选片、播放、收藏、继续观看和下载。
- 应用不内置任何默认视频源，首次使用必须先添加自己的采集接口。
- 当前主线体验围绕“浏览片库、查看详情、进入播放、回到上次观看位置”展开。
- Web 版通过 Vite dev/preview 本地代理请求用户配置的资源站，解决开发和本地预览时常见的 CORS 限制。

## 已实现功能

### 1. 数据源管理

- 支持添加自定义视频源，字段包含名称、网站地址、采集接口 URL 和接口格式。
- 支持 `auto`、`json`、`xml` 三种解析模式，其中 `auto` 会自动识别返回格式。
- 支持单独测试视频源可用性。
- 支持“测试并启用”，通过后保存视频源。
- 支持切换当前启用的视频源。
- 支持删除已保存的非内置视频源。
- 切换视频源前会重新验证接口，验证失败时保留当前状态。
- 已保存的视频源和当前启用源都会持久化到 localStorage。

### 2. 启动加载与本地缓存

- 应用启动后会优先恢复当前视频源对应的本地分类和首页预览缓存。
- 当本地已有缓存时，会先显示缓存内容，再按需请求在线数据。
- 应用会按视频源分别保存缓存，不同源之间互不干扰。
- 首页预览缓存会按小时周期刷新。
- 用户可以手动点击刷新，强制更新当前内容。

### 3. 首页与媒体库浏览

- 左侧导航提供“最新更新”“我的收藏”“继续观看”和分类入口。
- 默认首页是“最新更新”，使用本地预览优先的展示逻辑。
- 支持顶级分类浏览。
- 当某个顶级分类下存在子分类时，右侧顶部会显示子分类切换条。
- 父分类在预览模式下会聚合其子分类内容。
- 首页浏览区由焦点影片、继续观看横向入口、精选影片、全部影片几部分组成。
- 精选影片会优先把已收藏内容排在前面。
- 全部影片按批次展示，可反复“换一批”浏览。

### 4. 搜索与筛选

- 支持关键字搜索。
- 搜索入口适用于影片名、演员或一般关键词匹配。
- 搜索结果会进入独立的搜索结果模式。
- 搜索结果支持继续翻页加载。
- 分类浏览状态下可通过 `More` 打开筛选搜索面板。
- 筛选面板支持按类型切换。
- 支持按年份筛选。
- 支持按地区筛选。
- 支持重置筛选条件。
- 当前筛选选项是应用内置集合，不是从资源站动态拉取。
- 当资源站明确返回“不支持搜索”时，应用会降级为空结果展示。

### 5. 浏览交互

- 单击影片卡片会选中影片并刷新详情。
- 双击影片卡片会进入播放页。
- 鼠标悬停卡片时会显示“可双击播放”的视觉提示。
- 卡片支持悬浮详情预览，展示简介、主演、导演、更新时间和剧集信息。
- 浏览区会根据当前内容状态显示加载中、空状态或结果列表。

### 6. 影片详情

- 选中影片后会异步加载影片完整详情。
- 如果列表项本身已经包含播放字段，会优先复用，减少额外请求。
- 详情区展示海报、标题、评分、分类、地区、年份、语言、导演、主演、简介和更新备注。
- 支持收藏和取消收藏。
- 支持展示多个播放源。
- 支持按播放源切换剧集列表。
- 支持在详情页直接选择剧集开始播放。
- 当资源站返回下载字段时，详情页会显示下载分区。
- 若当前影片存在观看记录，详情页会显示“上次看到哪里”的提示。

### 7. 播放能力

- 支持剧集播放。
- 对 mp4、m4v、mov、webm 等普通直链，使用浏览器 `<video>` 播放。
- 对 m3u8 地址，使用 hls.js 播放，并通过本地代理改写常见分片地址。
- 对 `/share/` 一类网页播放地址，提供 iframe 播放和“打开网页播放器”入口。
- 支持上一集和下一集切换。
- 支持快退 15 秒和快进 15 秒。
- 打开影片时会优先尝试选择名称包含 `M3U8` 的播放源，否则回退到第一个可用源。

### 8. 继续观看

- 播放过程中会定时记录观看进度。
- 离开播放页时会再次记录当前进度。
- 记录内容包含影片、所属视频源、播放源、剧集、播放位置、时长和更新时间。
- 继续观看列表会按最近观看时间排序。
- 左侧提供独立“继续观看”入口。
- 首页也会展示最近播放内容的横向快捷入口。
- 再次进入同一影片时，应用会优先恢复到上次的播放源、剧集和时间点。
- 当观看已接近片尾时，进度会自动归零。
- 继续观看记录支持手动删除。
- 如果观看记录所属的视频源仍存在，应用可自动切回原视频源后继续播放。

### 9. 收藏

- 支持收藏影片。
- 支持取消收藏。
- 收藏记录会保存所属视频源信息，便于跨源恢复。
- 左侧提供独立“我的收藏”入口。
- 收藏页支持直接打开详情。
- 收藏页支持双击后直接继续播放。
- 如果收藏所属源与当前源不同，应用会优先切换回原视频源再打开影片。

### 10. 下载

- 当详情存在下载列表时，可直接点击剧集发起下载。
- 下载任务会显示在右下角下载浮层中。
- 浮层会展示任务标题、进度和当前状态。
- 下载完成后可再次触发浏览器保存。
- 下载文件由浏览器保存到用户默认下载位置。
- 当前下载逻辑更适合直链文件资源。

### 11. 应用级体验

- 主界面采用深色影院式布局，桌面端左侧导航，右侧内容区。
- 窄屏下布局会折叠为单列，保留主要操作入口。
- 浏览态和播放态在同一 Web 应用内切换。
- 应用整体使用统一的深色影院风格主题。
- 开发/预览环境使用本地代理请求媒体 API、解析网页播放候选地址并代理媒体内容。

## 操作说明

### 1. 首次使用

1. 运行 `npm run dev` 后打开本地地址。
2. 点击左下角“配置源”。
3. 在弹窗中填写名称、可选网站地址、采集接口 URL。
4. 接口格式通常可先选“自动”。
5. 先点击“测试”，确认接口能正常返回数据。
6. 测试通过后点击“测试并启用”。
7. 启用成功后，主界面会开始加载该视频源的分类和首页内容。

### 2. 浏览内容

1. 点击左侧“最新更新”进入首页。
2. 点击左侧任意分类可切换到对应片库。
3. 如果顶部出现子分类条，可继续点子分类缩小范围。
4. 在首页可通过“精选影片”和“全部影片”快速翻看当前资源。
5. 点击“换一批”可以切换当前展示批次。

### 3. 搜索与筛选

1. 在搜索框输入关键词并回车，可进入搜索结果页。
2. 如果想在某个分类下进一步缩小范围，点击右上角 `More`。
3. 在筛选面板中可切换类型、年份、地区。
4. 想恢复默认条件时，点击“重置”。

### 4. 查看详情与播放

1. 单击影片卡片，可刷新详情。
2. 双击影片卡片，可直接进入播放页。
3. 进入播放页后，可在“播放列表”中切换播放源和剧集。
4. 需要连看时，可使用“上一集”“下一集”按钮。

### 5. 继续观看

1. 正常播放后，应用会自动记录观看进度。
2. 下次可从左侧“继续观看”进入最近播放的影片。
3. 也可以直接在首页的“继续观看”横向栏点击恢复。
4. 若不再需要某条记录，可在“继续观看”页移除。

### 6. 收藏与下载

1. 在详情页点击“收藏”，即可把影片加入“我的收藏”。
2. 以后从左侧“我的收藏”可快速打开或继续播放。
3. 如果详情页出现“下载”分区，点击具体剧集即可开始下载。
4. 下载状态会显示在右下角浮层里。

## 当前限制

- 不内置任何默认可用的视频源。
- 静态部署时需要额外提供与 Vite 本地代理等价的代理能力，否则很多资源站会受 CORS 限制。
- 应用的搜索、播放和下载体验高度依赖用户配置的资源站接口质量。
- 当前筛选维度只有类型、年份、地区，没有独立的标签、语言、导演等高级筛选入口。
- 浏览器下载无法强制保存到固定文件夹，也无法直接在 Finder 中定位文件。
- iframe 网页播放器可能受资源站的跨域和嵌入策略限制。
- m3u8 分片、字幕、加密 key 和跨域资源可能需要资源站或代理配合。
